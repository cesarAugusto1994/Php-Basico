Web(Arquitetura Cliente-Servidor)

Cliente = Html, CSS, JavaScript Servidor = PHP, Java, Python, Ruby Um ambiente não entende o outro.

web é um dialogo entre estes dois lados(cliente-servidor) a maquina cliente faz uma requisicao, solicitando ao servidor um determinado arquivo, e o servidor obtem esse arquivo caso exista, processa o arqui se necessario e então entrega a resposta do arquivo ao cliente.

quando ocliente faz a solicitacao de um arquivo html, o servidor recebe a requisicao e procura em seu disco, ao encontrar, como o servidor não entende html por ser client-side então envia o arquivo a maquina cliente que por sua vez o browser compila o arquivo reinderizando na tela do usuario.

se fizer um requisição de um arquivo php, o servidor procura em seu disco, porém ao contrario do html onde ele apenas entrega o arquivo e o cliente tambem não entende php, no caso do arquivo php o arquivo é interpretado pelo servidor que gera um aquivo html para ser enviado ao cliente.
